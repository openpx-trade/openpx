## Order Types

Pass `order_type` in the `params` map when calling `create_order`. Defaults to `gtc` if omitted.

| Order Type | Kalshi | Polymarket | Opinion | Behavior |
|------------|:------:|:----------:|:-------:|----------|
| `gtc` | Yes | Yes | Yes (→ LIMIT) | Good-til-cancelled — rests on the book until filled or cancelled |
| `ioc` | Yes | Yes | Yes (→ MARKET_ORDER) | Immediate-or-cancel — fills what it can immediately, cancels the rest |
| `fok` | Yes | Yes | Yes (→ MARKET_ORDER) | Fill-or-kill — must fill entirely in one shot or is cancelled |

> **Note:** Opinion maps `ioc` and `fok` to its native `MARKET_ORDER` type. The partial-fill-then-cancel semantics of `ioc` are not precisely preserved — both behave as market orders.

## CLI Commands

The CLI (`openpx`) supports all read-only methods and WebSocket streams. Trading operations (`create_order`, `cancel_order`) are SDK-only.

| `fetch-market` | Yes | Yes | Yes |
