title: "Connecting"
description: "Open a WebSocket and subscribe to markets."

In Python and TypeScript, call `exchange.websocket()` to reuse credentials.

## Subscribe target

| Exchange | What to pass |
|----------|--------------|
| Polymarket | A CLOB token ID (e.g. `Market.token_id_yes` / `token_id_no`) |
| Kalshi | Market ticker (e.g. `KXBTC-25MAR14-T20000`) |

Polymarket also exposes `subscribe_pair(yes_token, no_token)` for binary
markets — both sides get full book updates and `price_change` events on
either side are dispatched correctly.

## Example

<CodeGroup>

```rust Rust
use openpx::WebSocketInner;
use px_core::websocket::OrderBookWebSocket;
use serde_json::json;

let mut ws = WebSocketInner::new("polymarket", json!({
    "private_key": std::env::var("POLYMARKET_PRIVATE_KEY")?,
    "funder": std::env::var("POLYMARKET_FUNDER")?,
}))?;

let updates = ws.updates().expect("first call");
let session = ws.session_events().expect("first call");

ws.connect().await?;
ws.subscribe("0x311d0c4b…").await?;
```

```python Python
ws = exchange.websocket()
ws.connect()
ws.subscribe("token-id-or-ticker")

for update in ws.updates():
    ...  # Snapshot / Delta / Trade / Fill
```

```typescript TypeScript
const ws = exchange.websocket();
await ws.connect();
await ws.subscribe("token-id-or-ticker");

await ws.onOrderbookUpdate("token-id-or-ticker", (err, update) => {
  if (err) return console.error(err);
  console.log(update);
});
```

</CodeGroup>

## Outcome labels (Polymarket)

To get `"Yes"` / `"No"` outcome labels on `Trade` and `Fill` events, either
subscribe via `subscribe_pair`, or call `register_outcomes(yes_token_id,
no_token_id)` before subscribing. No-op on Kalshi.
