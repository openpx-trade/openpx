OpenPX exposes a single multiplexed WebSocket per exchange. One connection
handles every subscribed market; consumers read from two streams.

| Stream | Type | Carries |
|--------|------|---------|
| `updates()` | `WsUpdate` | Per-market events: `Snapshot`, `Delta`, `Clear`, `Trade`, `Fill` |
| `session_events()` | `SessionEvent` | Connection-level: `Connected`, `Reconnected`, `Lagged`, `BookInvalidated`, `Error` |

Both streams are **single-consumer** — calling either method twice returns
`None` (Rust) or raises (Python/TS). To fan out, run one consumer that
re-broadcasts.

## States

`WebSocketState`: `Disconnected`, `Connecting`, `Connected`, `Reconnecting`,
`Closed`.
