OpenPX exposes a `Sports` handle backed by ESPN's public endpoints. The API
covers the catalog (sports, leagues, games), live game state, and ESPN-only
research data (teams, athletes, standings, news, schedule, play-by-play).

## Quickstart

<CodeGroup>

```rust Rust
use openpx::{Sports, GameFilter};

let sports = Sports::new()?;
let games = sports.list_games(GameFilter {
    league: Some("nfl".into()),
    ..Default::default()
}).await?;
```

```python Python
from openpx import Sports

sports = Sports()
games = sports.list_games(league="nfl")
```

```typescript TypeScript
import { Sports } from "openpx";

const sports = new Sports();
const games = await sports.listGames({ league: "nfl" });
```

</CodeGroup>

## Surface

| Group | Purpose |

| [ESPN Extensions](/api/sports/espn/teams) | Teams, athletes, standings, schedule, news, play-by-play — research data ESPN exposes that doesn't fit the catalog primitives. |
