OpenPX gives traders and quants a single, low-latency interface across multiple prediction markets. The Rust engine handles networking, parsing, and dispatch; thin language bindings expose the same surface to Python and TypeScript callers without rewriting business logic per language.

## Design goals

- **Speed first.** Every code path is tuned for latency. WebSocket frames are decoded with `simd-json` past a size threshold; HTTP clients share tuned defaults; hot loops avoid allocation.
- **Bring your own credentials.** OpenPX never holds keys for you. You construct an `Exchange` with a config dict and own the secret material end-to-end.
- **Unified, not lowest-common-denominator.** The `Exchange` trait covers what every supported venue actually supports; capability flags surface per-exchange differences instead of pretending they don't exist.
